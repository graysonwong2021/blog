關於這個網站的出現

I’m going to update this content before 2024/03/18.(Can’t be update if it expires)
Updated on 2024-03-18

前言

先說明一下，想要建立一個人部落格這個念想是從5年前（2019）已經存在，只不過當時卡在發佈上網的階段，而以自己當時的悟性無法獨立解決，按照網路上的教學文也總有一些地方出現報錯，結果我就默默的把這件事情放在一角冷卻了。

這個網站是用靜態網頁產生器SSG：Hexo ＋ Hexo Theme：NexT ＋ GitHub Page 所搭建出來的，因為是開源代碼，費用全免，還有各種add-on，非常值得一試。

那接下來，就Step by Step記錄一下整個構建流程（以防自己太容易忘記）：

01 搭建Hexo環境

這𥚃說的搭建其實就把大神設計好的代碼以指令方式下載下來 可參考官方的安裝指引 Hexo Doc

首先安裝Node.js/Git （有興趣再研究）
Node
Node.js 就是一個開源的運行環境（For JavaScript）
原來在Node.js出現之前，JavaScript只能活在瀏覽器，不能在Server中運行，當時的前端工程師要透過學習不同的程式語言才能管理後端的運作，而 Node.js 就應運而生，解決了問題。
Git
Git 是一個版本控制器，原來GitHub的是一個把Git伺服器視覺化的商業網站，但網站可讓大家透過Web介面來操作一些原本需要複雜的 Git 指令才能做到的事。除了GitHub，還一個Gitlab網站，與Github大同小異，差異有待研究。

簡單了解了Node以後，就可以打開Terminal終端機透過指命下載所需資料

這邊提一下NPM 是 Node Package manager 的簡稱，是管理 node 套件的工具

02 下載並套用主題

a. 從git上取主題

從git上複製NexT主題，地址後面的themes/NexT是指定存取位置，像這𥚃就是存在themes𥚃。

這𥚃要提一下，原來現在NexT有三個不同的源頭：

版本 年份 倉庫
v5.1.4 或更低 2014 - 2017 https://github.com/iissnan/hexo-theme-next
v6.0.0 ~ v7.8.0 2018 - 2019 https://github.com/theme-next/hexo-theme-next
v8.0.0 或更高 2020 https://github.com/next-theme/hexo-theme-next

原因是原創大神iissnan在2017就放下了NexT專案，後面由其他貢獻者繼續更新，我自己就選擇了持續有人維護的最新版本。

b. 在root folder（最外層資料來）上的_config.yml（不是theme folder𥚃的）找到 theme：的，把參數改成 NexT

最後再輸入 hexo s，就看到Next主題已經套上。

c. 修改 NexT 主題
發現有兩種方法，一種是直接修改theme folder𥚃的配置檔_config.yml。
另外一種是把theme folder𥚃的配置檔_config.yml複製至root folder（最外層資料來）上，並把其重新命名為_config.next.yml，再進行修改，原因是以防後續更新版本導致衝突。

修改裡面的 Scheme Settings，把想要的取消註解，其他全部註解掉（像我就是使用 Mist）。

URL：
－url 填自己的網域，有自己的網域的話就填自己的，沒有的話可以填 github page 的網址

03 發佈上線

a. 首先到 github 新開一個 Repository
然後取名叫做 username.github.io （username 是自己帳號的名字，如果要開更多網站的話可以取其他名字，但後面有地方要小改一點） 選 public，然後其他都不要打勾，建立專案repo

b. 把本地資料夾跟 Github 連上，輸入以下指令
